## 5. СТАТУСЫ ЗАДАЧ (КАНОНИЧЕСКИЕ)

### 5.1. Общие правила
- **Каждая задача обязана иметь статус.**
- Задача без статуса считается **несуществующей**.
- В один момент времени у задачи может быть **только один статус**.
- Переходы между статусами фиксируются кратко (1 строка).

---

### 5.2. Список статусов

#### NEW
**Что это:**  
Новая задача / входящий сигнал.  

**Условия:**  
- цель ещё не зафиксирована  
- решения нет  

**Запрет:**  
- не выполнять работу в статусе NEW  

---

#### READY
**Что это:**  
Задача понята и готова к выполнению.  

**Условия:**  
- цель зафиксирована  
- есть решение  
- определён следующий шаг  

---

#### WAIT
**Что это:**  
Ожидание внешнего фактора.  

**Условия:**  
- требуется ответ / документ / доступ  
- выполнение невозможно сейчас  

**Обязательно указывать:**  
- *что именно ждём*  
- *от кого*  
- *до какого момента*  

---

#### IN_PROGRESS
**Что это:**  
Активное выполнение.  

**Условия:**  
- назначен владелец  
- выполняется конкретный шаг  

**Ограничение:**  
- не более 1–2 задач одновременно в этом статусе  

---

#### DONE
**Что это:**  
Задача выполнена.  

**Условия:**  
- результат создан  
- артефакт существует  
- следующий шаг либо не нужен, либо зафиксирован  

---

#### ARCHIVE
**Что это:**  
Задача закрыта и убрана из активного поля.  

**Условия:**  
- результат не нужен сейчас  
- контекст может понадобиться в будущем  

---

#### DROP
**Что это:**  
Задача сознательно отменена.  

**Условия:**  
- зафиксирована причина отказа  
- к задаче не возвращаемся  

---

### 5.3. Разрешённые переходы

NEW → READY  
READY → IN_PROGRESS  
IN_PROGRESS → DONE  
IN_PROGRESS → WAIT  
WAIT → IN_PROGRESS  
DONE → ARCHIVE  

Любые другие переходы — **ошибка процесса** и требуют фиксации причины.
